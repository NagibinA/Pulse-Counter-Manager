# Прошивка для ESP8266 (Wemos D1 mini)

## Описание

Эта прошивка предназначена для сбора данных с импульсного счетчика электроэнергии (воды, газа, тепла) с помощью фототранзистора TEMT6000. Устройство подключается к WiFi, отправляет данные в MQTT и управляется командами из Home Assistant.

**Принцип работы:** Светодиод счетчика мигает с частотой, пропорциональной потребляемой мощности. Фототранзистор фиксирует вспышки, ESP8266 считает импульсы и публикует их в MQTT. Home Assistant через интеграцию Pulse Counter Manager преобразует импульсы в киловатт-часы и рубли.

## Подготовка Arduino IDE

### 1. Установите Arduino IDE

Скачайте с [официального сайта](https://www.arduino.cc/en/software)

### 2. Добавьте поддержку ESP8266

Файл → Настройки → Дополнительные ссылки для менеджера плат:

https://arduino.esp8266.com/stable/package_esp8266com_index.json


Инструменты → Плата → Менеджер плат → найдите `esp8266` → установите

### 3. Установите библиотеку PubSubClient

Скетч → Подключить библиотеку → Управлять библиотеками → найдите `PubSubClient by Nick O'Leary` → установите

### 4. Настройте библиотеку (ВАЖНО!)

Библиотеку PubSubClient нужно немного отредактировать. Найдите файл:

Windows: C:\Users*Username*\Documents\Arduino\libraries\PubSubClient\src\PubSubClient.h
Linux: ~/Arduino/libraries/PubSubClient/src/PubSubClient.h
macOS: ~/Documents/Arduino/libraries/PubSubClient/src/PubSubClient.h


Откройте его и **увеличьте значение MQTT_KEEPALIVE** с 15 до 120:

```cpp
#define MQTT_KEEPALIVE 120   // было 15, измените на 120
```
## Настройка прошивки
Скачайте файл pulse_counter_esp8266.ino

Откройте его в Arduino IDE

Найдите в начале файла секцию НАСТРОЙКИ и измените под себя:

// ========== НАСТРОЙКИ (ИЗМЕНИТЕ ПОД СЕБЯ) ==========
const char* ssid = "YourWiFi";              // Имя вашей WiFi сети
const char* password = "YourPassword";      // Пароль WiFi
IPAddress mqttServer(192, 168, 1, 100);     // IP адрес MQTT брокера
const char* clientID = "PulseCounter";      // Уникальное имя клиента
const char* mqtt_username = "";             // Логин MQTT (если нужен)
const char* mqtt_password = "";             // Пароль MQTT (если нужен)


Как узнать IP адрес MQTT брокера?

Если вы используете Home Assistant с надстройкой Mosquitto, IP адрес — это IP вашего Home Assistant. Обычно 192.168.1.xxx


## Прошивка Wemos D1 mini
Подключите Wemos D1 mini к компьютеру через USB-кабель

В Arduino IDE выберите плату: Инструменты → Плата → ESP8266 → LOLIN(WEMOS) D1 R2 & mini

Выберите порт: Инструменты → Порт → COMx (Windows) или /dev/ttyUSB0 (Linux)

Нажмите кнопку Загрузить (стрелка вправо)

Результат: Внизу окна появится надпись "Загрузка завершена".


Установка на счетчик
Обесточьте счетчик (отключите автомат) — для вашей безопасности!

Приклейте датчик TEMT6000 точно напротив светодиода счетчика

Зафиксируйте датчик черной изолентой (чтобы не попадал внешний свет)

Подключите Wemos D1 mini к питанию (USB-зарядка)

Включите автомат обратно

⚠️ Важно: Все работы в электрощитке проводите только при отключенном питании!

Проверка работы
Способ 1: Через MQTT клиент
Установите MQTT-клиент (например, MQTT Explorer) или используйте командную строку:

bash
# Подпишитесь на все топики счетчика
mosquitto_sub -h 192.168.1.100 -t "Counter/#" -v
Что должно появиться:

Counter/Available online — ESP подключена

Counter/day 0 (или Counter/night 0) — начальное значение

При мигании светодиода счетчика — числа увеличиваются

Способ 2: Через Home Assistant
После настройки интеграции Pulse Counter Manager вы увидите сенсоры:

sensor.energy_day — дневные показания

sensor.energy_night — ночные показания

sensor.pulse_counter_available — статус ESP

Управление счетчиком
Отправляйте команды в топик Counter/choice:

Команда	Действие
day	Переключение на дневной тариф
night	Переключение на ночной тариф
+	Увеличить порог срабатывания на 10
-	Уменьшить порог срабатывания на 10
Пример через командную строку:

bash
mosquitto_pub -h 192.168.1.100 -t "Counter/choice" -m "day"
Пример через Home Assistant:

yaml
service: mqtt.publish
data:
  topic: "Counter/choice"
  payload: "night"
Настройка порога чувствительности
Если ESP не видит импульсы (светодиод мигает, а счетчик не растет):

Отправьте команду - (уменьшить порог)

Повторяйте, пока счетчик не начнет реагировать

Если ESP ловит лишние импульсы (счетчик растет без мигания светодиода):

Отправьте команду + (увеличить порог)

Повторяйте, пока ложные срабатывания не прекратятся

Оптимальный порог обычно находится в диапазоне 30-80.

Устранение неполадок
ESP не подключается к WiFi
Проверьте правильность имени и пароля в настройках

Убедитесь, что WiFi работает на частоте 2.4 ГГц (ESP8266 не поддерживает 5 ГГц)

Попробуйте перезагрузить роутер

ESP не подключается к MQTT
Проверьте IP адрес брокера (ping 192.168.1.100)

Проверьте логин и пароль

Убедитесь, что порт 1883 открыт

Счетчик не видит импульсы
Проверьте, что светодиод на счетчике действительно мигает

Уменьшите порог командой -

Проверьте, что датчик направлен точно на светодиод

Ложные срабатывания
Увеличьте порог командой +

Защитите датчик от внешнего света черной изолентой

Пример использования в Home Assistant
После установки интеграции Pulse Counter Manager:

MQTT брокер уже настроен в интеграции

Сенсоры создаются автоматически

Тарифы переключаются по времени (настраивается в интеграции)

Никаких дополнительных автоматизаций писать не нужно! Интеграция всё делает сама.

Дополнительно
Сброс счетчика
Если нужно обнулить показания (например, после замены счетчика):

В Home Assistant найдите сенсор sensor.energy_day

Нажмите на три точки → Настройки

Измените значение

Или через сервис:

yaml
service: pulse_counter.set_day_kwh
data:
  value: 0
