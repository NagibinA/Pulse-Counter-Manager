## ✨ Возможности

| Тип счетчика | Поддержка | Особенности |
|-------------|-----------|-------------|
| ⚡ **Электроэнергия** | ✅ Полная | Двухтарифный учет (день/ночь), автоматическое переключение |
| 💧 **Вода** | ✅ Полная | Однотарифный учет |
| 🔥 **Газ** | ✅ Полная | Однотарифный учет |
| 🌡️ **Тепло** | ✅ Полная | Однотарифный учет |

### Основные функции

- 📊 **Автоматический расчет стоимости** — по дневным и ночным тарифам
- 📤 **Экспорт показаний** — в отдельные MQTT топики (с поддержкой отдельного брокера)
- 💾 **Сохранение состояния** — при перезагрузке Home Assistant
- 🔧 **Настройка чувствительности ESP** — через MQTT команды (+/-)
- 📈 **Сенсор скорости импульсов** — отображение имп/мин
- 🔗 **Ссылка на актуальные тарифы** — в настройках интеграции

## 📥 Установка

### Через HACS (рекомендуется)

1. Откройте HACS → Интеграции
2. Нажмите на три точки (⋮) → **Custom repositories**
3. Вставьте URL: `https://github.com/NagibinA/Pulse-Counter-Manager`
4. Выберите категорию: **Integration**
5. Нажмите **Добавить**
6. Нажмите **Скачать** (Download)
7. Перезапустите Home Assistant

### Вручную

```bash
cd /config/custom_components
git clone https://github.com/NagibinA/Pulse-Counter-Manager.git pulse_counter
```

Перезапустите Home Assistant.

## 🛠️ Настройка ESP

### Вариант 1: Arduino прошивка

1. Откройте `firmware/pulse_counter_esp8266.ino` в Arduino IDE
2. Настройте параметры:

```cpp
const char* wifi_ssid = "YourWiFi";
const char* wifi_password = "YourPassword";
IPAddress mqtt_broker(192, 168, 1, 100);
```

3. Загрузите на Wemos D1 mini
4. Приклейте TEMT6000 напротив светодиода счетчика

### Схема подключения TEMT6000

| TEMT6000 | Wemos D1 mini |
|----------|---------------|
| VCC (средний) | 3.3V |
| GND | GND |
| OUT (крайний) | A0 |

> 💡 **Совет:** Защитите фототранзистор от внешнего света.

## ⚙️ Настройка в Home Assistant

**Шаг 1:** Настройки → Устройства и интеграции → Добавить интеграцию → Pulse Counter Manager

**Шаг 2:** Введите параметры MQTT брокера (IP, порт, логин/пароль)

**Шаг 3:** Добавьте счетчик, заполнив параметры в секциях:
- MQTT топики
- Тарифы (для электричества)
- Параметры счетчика
- Начальные показания
- Экспорт (опционально)

## 📊 Сенсоры

### Электроэнергия — основные

| Сенсор | Ед. изм. | Описание |
|--------|----------|----------|
| `{name} Энергия дневная` | кВт·ч | Дневное потребление |
| `{name} Энергия ночная` | кВт·ч | Ночное потребление |
| `{name} Стоимость электричества за месяц` | руб | Стоимость за месяц |

### Диагностические

| Сенсор | Ед. изм. | Описание |
|--------|----------|----------|
| `{name} Всего` | кВт·ч | Общее потребление |
| `{name} Получено импульсов` | имп/мин | Скорость импульсов |

### Вода/Газ/Тепло

| Сенсор | Ед. изм. | Описание |
|--------|----------|----------|
| `{name} Всего` | м³/Гкал | Общее потребление |
| `{name} За месяц` | м³/Гкал | Потребление за месяц |
| `{name} Стоимость` | руб | Стоимость за месяц |

## 🎮 Сервисы

```yaml
# Для электроэнергии
service: pulse_counter.set_day_kwh
data:
  value: 123.45

service: pulse_counter.set_night_kwh
data:
  value: 67.89

# Для воды/газа/тепла
service: pulse_counter.set_total_value
data:
  value: 45.67

# Сброс месячных показаний
service: pulse_counter.reset_monthly
```

## 🔧 Устранение неполадок

**ESP не видит импульсы** — отправьте команду `-` в топик `Counter/choice` (уменьшить порог)

**Ложные срабатывания** — отправьте команду `+` (увеличить порог)

**Проверка статуса ESP:**
```bash
mosquitto_sub -h broker_ip -t "Counter/Available"
```

**Ошибка MISSING_VALUE** — перезагрузите HA и очистите кэш браузера
